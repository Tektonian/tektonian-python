# Contributing to Simulac
You!, who are reading this page.
If you're reading this page, you have probably encountered an issue or are interested in contributing to the project.
First and foremost, thanks for your interest in *Simulac*.

# Asking question
If you have a simple question about using Simulac or Tektonian services, please visit our [Discord channel](https://discord.gg/zbSaU8ZbS), instead of opening an issue. (e.g., What's project roadmap, how-to-do to fullfil ~~thing)

Our developers will do their best to answer your questions there.

However, if you encounter an issue with the **Simulac** project itself, please open an issue and describe it clearly. (e.g., bugs, errors)

GitHub issues should stay focused on bugs, missing documentation, and concrete project improvements.

A well-organized question can also be a useful resource for others looking for help.

## Reporting issues
Before you create a new issue, please search existing issues on Github to see whether the same issue has already been reported.

When you create an issue, try to include:

- a short summary of the problem
- steps to reproduce it
- expected behavior
- actual behavior
- your environment including OS, Python version, installationi method
- logs, screenshots, or stack traces when applicable

A well-written issue helps us respond faster and makes the thread useful to other users later.

# Code contribute
First of all, we **DO NOT** accept any AI-generated codes.

Tektonian team believes that good code is an explainable code.
If you do not fully understand the code you submit and cannot clearly explain its intend or purpose, adding it to the project only creates technical debt for others.
Your interest in this project and contributions all help this project improve.

We sincerely appreciate the time and effort you have given to this project.

Thank You!
